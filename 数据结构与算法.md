# 数据结构基本概念

数据：数据是信息的载体,是描述客观事物属性的数，字符及所有能输入到计算机中并被计算机程序识别和处理的符号的集合。数据室计算机程序加工的原料  
数据元素：数据元素是数据的基本单位，通常作为一个整体进行考虑和处理，一个数据元素可由若干数据项组成，数据项是构成数据元素的不可分割的最小单位。    
结构：各个元素之间的关系  数据结构：数据结构是相互之间存在一种或多种特定关系的数据元素的集合  
数据对象：数据对象是具有相同性质的数据元素的集合，是数据的一个子集  
数据结构三要素：    逻辑结构：集合 线性结构 树形结构  图形结构        线性结构：元素之间一对一关系，除了第一个元素，所有的元素都有唯一前驱，除了最后元素，所有元素都有唯一后驱。      树形结构：数据元素之间是一对多的关系。  图形结构：多对多  
                     物理结构（存储结构）：如何用计算机表示出数据元素的逻辑结构。 
                                           顺序存储：把逻辑上相邻的元素存储在物理位置也相邻的存储单元中。各个元素在物理上是连续的   
                                           链式存储：逻辑上相邻的元素在物理位置上可以不相邻，借助指示元素位置的指针来表示数据元素之间的逻辑关系。    
                                           索引存储：在存储元素信息同时，建立附加索引表，索引表中每一项称索引项，索引项的一般形式为(关键字，地址)关键字来区分不同的数据元素  
                                           散列存储：根据元素关键字直接计算出钙元素存储地址，又称哈希存储。  
                                           数据的存储结构会影响存储空间分配的方便程度，还会影响对数据运算的速度。  
                     数据的运算：施加在数据上的运算包括运算的定义和实现。 定义是针对逻辑结构的，支出运算的功能；实现是针对存储结构的，指出运算的具体操作步骤。     
数据类型： 是一个值的集合和再次集合上的一组操作的总称       
          原子类型：其值不可再分的数据类型  
          结构类型：其值可以在分解为若干分量的数据类型。  
抽象数据类型：ADT是抽象数据组织及与之相关的操作。  

# 线性表与数组（字符串与压缩存储）
线性表是n个相同数据类型的数据元素的有限序列 n>=0;n为表长，如果n=0，空表，a1是表头，an是表尾，除了第一个元素，所有的元素都有唯一前驱，除了最后元素，所有元素都有唯一后驱    
线性表的基本操作：InitList(&L) 初始化表，构造一个空的线性表L,分配内存空间  
                  DestroyList(&L)销毁操作。销毁线性表，并释放线性表L所占用的内存空间。  
                  ListInsert：插入  
                  ListDelet：删除  
                  LocateElem(L,e)按值查找  
                  GetElem(L,i)按位查找  
                  对数据结构的操作：创建销毁 增删改查    
        顺序表：用顺序存储方式实现线性表的顺序存储，把逻辑上相邻的元素存储在物理存储也相邻的存储单元中。       
              实现：静态分配，初始就定义了最大长度，顺序表的大小是固定的    
                    动态分配：malloc，L.data = (int) malloc(sizeof(int) * InitSize) InitSize = 10;  
              特点：可以在O(1)时间找到第i个元素  
                    拓展容量不方便，  
                    插入删除不方便，需要移动大量元素  
        链表：用链式存储的存储结构来实现一个线性表，是一种线性表，但是不会按照顺序结构来存储元素，链表是通过一组任意的存储单元来存储线性表中的数据元素的。对于每个数据元素除了要存储自身的信息外，还要存放下一个元素的地址，这两部分信息组成一个结点。  
              各个结点的先后关系用一个指针来表示  
              两种实现：带头节点：  
                        不带头结点：空表    插入和删除 第一个结点会比较麻烦。  
              typedef 关键字 重命名某种类型， 比如 typedef int zhengshu  就可以 zhengshu i  = 1;    
              查找：按位查找和按值查找  平均和最坏时间复杂度都为O(n)  
              单链表简历：尾插法，头插法：初始化一个单链表，指定节点的后插操作。    
      
              双链表：每个节点有两个指针，指向前驱和后驱，  
                      遍历：后向遍历  前向遍历，   时间复杂度O(n)    
              循环单链表：单链表最后的结点的next指针指向头结点。    
              循环双链表：  
              静态链表： 分配一整片连续的内存空间，各个结点集中安置，指向下个元素的地址改变为下个元素的游标，类似于数组。  
        顺序表与链表的比较：逻辑上都是线性的，存储结构不同  
                            销毁：顺序表若是静态分配 系统自动回收，修改length=0，若是动态分配malloc，则需要手动free。  
                            插入删除：顺序表需要将后续元素进行后移或者前移，如果数据元素本身占用很大，那么这样会很慢，链表则是先找到，然后插入或删除即可，时间复杂度两者都是On但是后者明显更快。  
        数组：由相同类型元素组成，并且使用一块连续的内存来存储，可以吧一个一维数组当做一个线性表，可以吧二维数组看作是数据元素是一维数组所组成的一维数组的线性表。数组的行列固定后，可以通过一个映像函数，根据下标得到它的存储结构。  
        字符串：是一种特殊的线性表，它的数据元素仅由一个字符组成，同时字符串是由零个或多个任意字符组成的字符序列。通常采用顺序存储的方法  
              模式匹配：在字符串中寻找子串   
        压缩存储：高级语言里面，我们使用二维数组来存储矩阵元素，但是矩阵中有许多值相同的元素或者零元素，使用二维数组来保存可能会浪费空间，因此牵扯出矩阵的压缩存储问题。    
                  所谓的压缩存储：为多个值相同的元只分配一个存储空间，对零元不分配存储空间  
                  矩阵分为稀疏矩阵和特殊矩阵 前者：零元素的数量远多于非零元素数量，并且非零元素的位置没有规律可言。      后者：值相同的元素或零元素在矩阵的分布中有一定规律  
                  特殊矩阵的压缩存储：根据其规律将其压缩到一维数组中，并且找到每个非零元素在一维数组中的对应关系  
                  稀疏矩阵：只存储非零元素，即用三元组来保存非零元素，可以前两元是位置，第三元是数据。
